# P0 角色库验收清单（Liora ≥ 0.6.1）

目标：角色像完整产品能力——**可存档、可默认、会话级切换、回复可归因**。

## 环境

- [ ] 桌面版 **≥ 0.6.0**
- [ ] 引擎就绪；至少一个模型
- [ ] **新建会话** 再测切换（避免与旧消息混淆）

## A. 角色库 CRUD（独立「角色库」页）

1. 右侧角色卡点 **打开角色库**（或标题旁「角色库」）
2. 库内 **新建角色**：名称 + 描述（可选系统指令）
3. [ ] 列表出现该角色；重启应用后仍在
4. [ ] **编辑** 改描述后保存，右侧英雄卡与库内卡片更新
5. [ ] **删除** 自定义角色成功；内置 Liora **无法删除**
6. [ ] **导出 / 导入 JSON** 可用

## B. 默认 vs 会话切换

1. 将自定义角色设为 **默认**（★）
2. [ ] **新建会话** 使用该默认角色（标题栏 / 右侧卡一致）
3. 在当前会话切换回 Liora
4. [ ] 仅当前会话变；其他会话角色不变
5. [ ] 再新建会话仍用「默认」角色（不是刚才切换的临时选择）

## C. 注入与标注

1. 自定义角色写明确人设（如「只用三字短句回答」）
2. 发一轮消息
3. [ ] 回复大致符合人设（模型能力允许范围内）
4. [ ] 气泡顶部显示角色名；底部有「回复角色：…」
5. 中途切换另一角色再发一轮
6. [ ] **历史气泡**仍标旧角色；**新气泡**标新角色

## D. 备份

1. 设置 → 导出备份
2. [ ] JSON 含 `characters` 数组
3. 改/删某角色后 **合并导入** 同一备份
4. [ ] 角色库恢复合理；旧备份无 `characters` 时本地角色不丢

## E. 产品原则抽查

1. [ ] 无 Reply style 芯片（主界面 / 设置）
2. [ ] 获取模型 → 导入 GGUF **无** System prompt 输入框
3. [ ] 系统侧不强制「勿拟人 / 勿宣称情感」类说教（人设以角色卡为准）

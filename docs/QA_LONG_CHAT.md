# P0 长聊验收清单（Liora）

目标：在 **8K 上下文**、记忆 **开启** 下，验证「能长聊」且「重点还在」。

## 环境

- [ ] 桌面版 **≥ 0.6.1**（0.5.1+ 亦可测长聊核心）
- [ ] 引擎就绪；至少一个模型（推荐 `qwen2.5:7b` 或本机已有模型）
- [ ] 设置：上下文 **8K**、篇幅 **正常**、记忆 **开**
- [ ] **新建会话** 再测（避免旧上下文干扰）

## A. 不断线（40～50 轮）

1. 用短句连聊 **40 轮以上**（可复制：「继续，用三句话」）
2. 观察顶部 **上下文用量**：
   - [ ] **冷区** 数字会随轮次上升（> 0）
   - [ ] **组装** token 不应单调顶满到 100% 后仍暴涨
   - [ ] 发送不应频繁失败 / 空回复
3. 可选：中途点 **立即整理记忆**，确认 toast 出现、不崩溃

**通过标准**：50 轮内可继续发送；无系统性「引擎挂了 / 502」；组装占用多数时间 < 90%。

## B. 仍记得住（名字 / 宠物）

1. 第 1～2 轮明确说：  
   `我叫小明，养了只叫豆豆的柯基。请记住。`
2. 再聊 **15～30 轮**无关话题，直到用量条 **冷区 ≥ 4** 或已整理记忆
3. 打开 **记忆中心**：
   - [ ] 可见名字或宠物相关 L3
4. 再问：`我叫什么？豆豆是什么动物？`
   - [ ] 回答基本正确（允许措辞不同）

**通过标准**：热区滚出后，仍能从记忆/摘要答对身份事实。

## C. 首启向导

1. 清空设置中的 `onboardingDone`（或新装 + 删 settings 后首次打开）
2. [ ] 出现三步：引擎 → 模型 → 数据目录
3. [ ] 引擎未就绪时无法点「下一步」
4. [ ] 无模型时无法进入第三步（或提示去获取模型）
5. [ ] 完成后不再自动弹出

## D. 错误文案（抽查）

| 场景 | 期望 |
|------|------|
| 引擎关着点发送 | 提示启动引擎，而非生硬英文堆栈 |
| 无模型 | 提示「获取模型」 |
| 代理导致 502/403 | 提示直连 127.0.0.1 / 关闭代理 |
